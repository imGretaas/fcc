Intro

Hypertext Markup Language (HTML) is the standard markup language for documents designed to be displayed in a web browser. It can be assisted by technologies such as Cascading Style Sheets and scripting languages such as JavaScript.

Web browsers receive HTML documents from a web server or from local storage and render the documents into multimedia web pages. HTML describes the structure of a web page semantically and originally included cues for the appearance of the docoument.

HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects such as interactive forms may be embedded into the render page.
HTML provides a means o creeate structured documents by denoting structural semantics for text such as headings, paragraph, lists, links, quotes and other elements.

HTML elements are delineated by tags, written using angle brackets.
Tags such as <img> and <input> directly introduce content into the page. Other tags such as <p> surrounded and provide informatio about document text and may include other tags as sub-elements.
Browsers do not display the HTML tags, but use them to interpret the content of the page.

HTML can embed programs written in a scripting language such as JavaScript, which affects the behavior and content of web pages. Inclusion of CSS degines the look and layout of content.
The World Wide Web Consortium (W3C), former maintainer of the HTML and current maintainer of the CSS standards, has encouraged the use of CSS over explicit presentational HTML since '97.

HTML Versions

There are many versions of HTML:

  • HTML 2
  • HTML 3
  • HTML 4
  • HTML 5

First version of HTML (HTML 2) was published in November 1995.
The second one was published as a W3C Recommendation in January 1997. It was the first version develiped and trandarized exclusively by the W3C, as the IETF (Internet Engineering Task Force) had closed its HTML Working Group on September '96.

HTML 4 was published as a W3C Recommendation. IT offers three variations:

  • Strict, in which deprecated elements are forbidden,
  • Transitional, in which deprecated elements are allowed,
  • Frameset, in which mostly only frame related elements are allowed.

Since December 1997, when HTML 4 was released, to May 2000 there were another three released that improved the language and respected ISO/IEC international standard.

The last one version of HTML was released in October 2014, that version is the current in use.
HTML 5 was upgraded three times, the last one was for HTML 5.2 in December 2017.

The first draft version of HTML was made in October 1991, it was called HTML Tags which was an informal CERN document listing 18 HTML tags and it was the first mentioned in public about the language.

An important mention is about XHTML; it is a separate language, made in the early 2000, that began as a reformulation of HTML 4.01 using XML 1.0. It is no longer being developed as a separate standard.

Markup

HTML markup consists of several key components, including those called tags, character-based on data types, characteer references and entity references. HTML tags most commonly come in pairs like <h1> and </h1>, although some represent empty elements ans so are unpaired, for example <img>.
The first tag in such a pair is the start tag and the second is the end tag (they are also called opening tags and closing tags).

Block and Inline Elements

There are two types of elements in HTML

  • Block Elements,
  • Inline Elements.

To follow some example of block elements

  • <div>
  • <h1> ... <h6>
  • <hr/>
  • <p>
  • <table>
  • <form>

To follow some example of inline elements

  • <dl>
  • <li>
  • <ol>
  • <ul>

A line-level element remain in line with the rest of the content instead a block-level element always starts on a new line and takes up the full width available.

Styling

If you want to make your pages great aestherically, you need to use another languange like CSS.
Cascading Stle Sheets (CSS) is a style sheet language used for describing the presentation of a document written in markup language like HTML.

CSS is designed to enable the separation of presentation and content, includin layout, colors and fonts. This separation can improve content accessibility, provide more flexibility and control in the specification of presentation characteristics, enable multiple web pages to share formatting by specifying the relevant CSS in a separatee .css file which reduces complexity and repetition in the structural content as well as enabling the .css file to be cached to improve the page load speed between the pages that the file and its formatting.

Separation of formatting and content also makes it feasible to present the same markup page in different styles for different rendering methods, such as on-screen, in print, by voice and on Braille-based tactile devices. CSS also has rules for alternate formatting if the content is accessed on a mobile device.

The name cascading comes from the specified priority scheme to determine which style rule applies if more than one rule matches a particular element. This cascading priority scheme is predictable.

The CSS specifications are maintained by the World Wide Web Consortium (W3C). The W3C operates a free CSS validation service for CSS documents.
